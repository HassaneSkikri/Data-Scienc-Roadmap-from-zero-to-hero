## what is git?

```
Git is a distributed version control system for tracking changes in source code during software development.
```

## what is github?

```
GitHub is a web-based platform used for version control. Git simplifies the process of working
with other people and makes it easy to collaborate on projects.
```

## what is version control?

```
Version control is a system that records changes to a file or set of files
over time so that you can recall specific versions later.
```

## what is a repository?

```
A repository is a central file storage location. It is used by version control systems to store multiple versions of files.
```

## what is a commit?

```
A commit, or "revision", is an individual change to a file (or set of files).
```

## what is a branch?

```
A branch is a parallel version of a repository. It is contained within the repository,
but does not affect the primary or master branch allowing you to work freely without disrupting the "live" version.
```

## what is a pull request?

```
Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub.
Once a pull request is opened, you can discuss and review the potential changes with collaborators 
and add follow-up commits before your changes are merged into the base branch.
```

## what is a merge?

```
Merging takes the changes from one branch (in the same repository or from a fork), and applies them into another.
```

## what is a conflict?

```
A conflict occurs when two separate branches have made edits to the same line in a file,
or when a file has been deleted in one branch but edited in the other.
```

## what is a merge conflict?

```
A merge conflict is an event that occurs when Git is unable to automatically resolve differences in code between two commits.
```

## what is git pull?

```
git pull is a Git command used to update the local version of a repository from a remote.
```

## what is git push?

```
git push is a Git command used to upload local repository content to a remote repository.
```

## what is git clone?

```
git clone is a Git command line utility which is used to target an existing repository and create a clone,
or copy of the target repository.
```

## what is git status?

```
git status is a command that displays the state of the working directory and the staging area.
```

## what is git add?

```
git add is a Git command used to add file contents to the index.
```

## what is git commit?

```
git commit is a Git command used to save the changes to the local repository.
```

## what is git branch?

```
git branch is a Git command that allows you to create, list, rename, and delete branches.
```

## what is git checkout?

```
git checkout is a Git command used to switch between branches in a repository.
```

## what is git merge?

```
git merge is a Git command used to merge the changes from one branch into another.
```

## what is git pull request?

```
git pull request is a command that lets you tell others about changes you've pushed to a branch in a repository on GitHub.
```

## what is git push?

```
git push is a Git command used to upload local repository content to a remote repository.
```

## what is git remote?

```
git remote is a Git command that shows all the remote repositories that are associated with a local repository.
```

## what is git log?

```
git log is a command that is used to list the version history of the current branch.
```

## what is git reset?

```
git reset is a Git command used to reset the index and working directory to the last state of the repository.
```

## what is git revert?

```
git revert is a Git command used to undo a previous commit.
```

## what is git rebase?

```
git rebase is a Git command used to reapply commits on top of another base trip.
```

## what is git tag?

```
git tag is a Git command used to create, list, delete or verify a tag object signed with GPG.
```

## what is git fetch?

```
git fetch is a Git command used to retrieve new work from a remote repository.
```

## what is git stash?

```
git stash is a Git command used to temporarily store changes in the working directory.
```

## what is git blame?

```
git blame is a Git command used to display the author of each line of a file.
```

## what is git diff?

```
git diff is a Git command used to display the differences between two versions of a file.
```

## what is git grep?

```
git grep is a Git command used to search the codebase for a specific string or pattern.
```

## what is git clean?

```
git clean is a Git command used to remove untracked files from the working directory.
```

## what is git cherry-pick?

```
git cherry-pick is a Git command used to apply a specific commit from one branch to another.
```

## what is git bisect?

```
git bisect is a Git command used to find the commit that introduced a bug by using binary search.
```

## what is git reflog?

```
git reflog is a Git command used to manage a list of commits that are referenced in the repository.
```

## what is git submodule?

```
git submodule is a Git command used to add a reference to another repository inside a Git repository.
```

## what is git worktree?

```
git worktree is a Git command used to manage multiple working trees associated with a single repository.
```

## what is git lfs?

```
git lfs is a Git extension used to manage large files and binary files in a Git repository.
```

## what is git hooks?

```
git hooks are scripts that Git executes before or after events such as commit, push, and receive.
```

## what is git archive?

```
git archive is a Git command used to create a tar or zip archive of the contents of a repository.
```

## what is git revert?

```
git revert is a Git command used to undo a previous commit.
```
